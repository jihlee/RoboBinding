Binding to custom views

RoboBinding allows you to add binding support for your own custom UI components. Simply implement the BindableView interface and type-parameterize it to the class you are implementing.

Here is an example taken from the sample project available on GitHub:

{% highlight java %} public class NavigationBar extends RelativeLayout implements BindableView { private Button leftButton; private Button rightButton; public NavigationBar(Context context, AttributeSet attrs) { super(context, attrs); ... } @Override public void mapBindingAttributes( CustomBindingAttributeMappings mappings) { mappings.mapCommandAttribute(leftButton, OnClickAttribute.class, "leftButtonAction"); mappings.mapCommandAttribute(rightButton, OnClickAttribute.class, "rightButtonAction"); } } {% endhighlight %}

The BindableView interface requires you to implement a single method, mapBindingAttributes(). On the CustomBindingAttributeMappings instance that is passed into this method you define the binding names that you want to expose to your users.

In this case we are exposing two bindings, 'leftButtonAction' and 'rightButtonAction'.

The CustomBindingAttributeMappings interface itself looks like this:

{% highlight java %} public interface CustomBindingAttributeMappings extends BindingAttributeMappings { void mapPropertyAttribute(S alternateView, Class> propertyViewAttributeClass, String attributeName); void mapCommandAttribute(S alternateView, Class> commandViewAttributeClass, String attributeName); void mapGroupedAttribute(S alternateView, Class> groupedViewAttributeClass, String... attributeNames); } {% endhighlight %}

As you can see, there are three types of bindings you can expose: Properties, Commands or Grouped binding attributes.
